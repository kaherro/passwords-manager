# passwords-manager

![Демонстрация](demo.gif)
Password Manager — консольное приложение для хранения паролей с простым интерфейсом на `ncurses`, локальной базой данных `SQLite` и элементами RSA-шифрования.

## Описание

Это приложение позволяет:
- создать публичный ключ, который используется для шифрования и восстановления паролей;
- добавить собственный пароль;
- просмотреть все сохранённые пароли после ввода публичного ключа;
- сгенерировать случайный пароль длиной 20 символов.

Пароли сохраняются в локальной базе `passwords_manager.db`.

## Возможности

- `Check my passwords` — открыть список всех паролей и расшифровать их с помощью введённого публичного ключа.
- `Generate a new key` — сгенерировать новый публичный ключ RSA.
- `Generate a new password` — создать случайный пароль из букв, цифр и специальных символов.
- `Add your own password` — добавить пароль вручную.
- `Exit password manager` — выйти из приложения.

## Требования

- C++17
- CMake 3.10+
- `ncurses` (`sudo apt install libncurses-dev`)
- `SQLite3` (`sudo apt install libsqlite3-dev`)

## Сборка

```bash
mkdir build
cd build
cmake ..
make
```

## Запуск

```bash
./password_manager
```

## Использование

1. Запустите приложение.
2. Навигируйте стрелками вверх/вниз.
3. Подтверждайте выбор клавишей `Enter` или стрелкой вправо.
4. Для работы с паролями требуется ввести публичный ключ RSA.
5. При добавлении пароля задайте заголовок и сам пароль.
6. Результат сохраняется в `passwords_manager.db`.

## Структура проекта

- `main.cpp` — основной интерфейс и логика меню.
- `src/BigInt.cpp` — реализация длинной арифметики для RSA.
- `src/rsa.cpp` — генерация ключей, шифрование и дешифрование.
- `src/db_manager.cpp` — работа с SQLite-базой.
- `include/` — заголовочные файлы.

## Ограничения и особенности

- Заголовок пароля ограничен 15 символами.
- Пароль вручную можно вводить до 20 символов.
- Публичный ключ должен состоять только из цифр.
- При потере ключа восстановление паролей будет невозможно.

## Советы

- Сохраните сгенерированный публичный ключ сразу после создания.
- Для генерации безопасных паролей используйте пункт `Generate a new password`.
- Файл базы `passwords_manager.db` создаётся автоматически при первом запуске.
